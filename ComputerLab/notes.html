  • On the teacher's computer, open the manager. Go to the Authentication tab.
    • Uncheck ACL-based logon authentication
    • Click Launch key file assistant
      • Create new access keys

Batch Install

  • Download all the programs you want to install.
  • Figure out how to have an unattended install, writing a batch script (see above).
  • Copy all the files and the batch script to the server.
  • Share the directory you put them in as \\server\Software\, making sure that both the NTFS permissions allows Everyone to read and the Share permissions as well.
  • Create a GPO that sets the startup script as the network path to your shared batch script (and whatever other settings you want, e.g. not sleeping, auto login registry tweaks, etc.)
  • Using iTALC, run "gpupdate /force" on all the machines. Then reboot them all.
  • On boot it should run the script.

  • Computer → Properties → Advanced → Profile Settings → [Default] → Copy To \\server.yapsda.org\Profiles\student

Deploying Windows

Switching from 192.168.1.x to 10.0.0.x:

  • Change the static IPv4 settings for the server (IP 10.0.0.103, Mask 255.255.255.0, Gateway 10.0.0.1, DNS 10.0.0.103). Press okay and everything.
  • Under DHCP create a different scope with the new IP ranges, making sure that 10.0.0.103 isn't in the assigned range. Copy all settings from the other scope (manually looking through)
  • Under DNS change the yapsda.org _msdcs IP address to be 10.0.0.103

Note you'll have to activate Windows somehow.

Steps to deploy

  • Copy the "To Install" directory from the server's Documents\Software\ to your flash drive (or go to \\server\Software to find the files when you need them later)
  • Plug the computer into the switch with the server connected, powered on, and running Windows Deployment Services
  • If you're locked out of the BIOS due to it being password protected, remove the PSWD pin on the Optiplex 380 computers (or probably something similar on the other computers)
  • Enable PXE booting in the BIOS (different for every computer; note on the acer computers: you have to enable LAN Option ROM under Integrated Peripherals to get PXE booting to show up under the boot menu)
  • Reboot, press F12, select PXE boot
  • Windows will boot. Select "My language is English"
  • Select to install Windows 7 Professional (or the customized image with Office and everything else preinstalled) and then on whatever partition you want (probably the second, largest one if there are two listed)
  • When it asks for a license key, uncheck the box that says activate with Internet, click Next without typing in a license key.
  • It'll ask for a username and password. User: sda, Password: InsertPasswordHere, Hint: ask the admin. If it says "sda" already exists, then just create sda2 or something. This account was already added to the image.
  • Set the timezone to whatever (Pacific is the default)
  • When it finally logs in, it'll come up with the student account and ask for what type of network this is. Click "Home."
  • Logout and login as the teacher
  • Activate Windows
  • Wait for all the applications to download and install (if you didn't install the customized image). Visual Studio will take a long time. Note that it's also installing Windows 7 SP1, so it'll reboot. Just leave it up until it reboots and finishes installing SP1.
  • If this is one for the lab, go look at the "Configuring iTALC" on this page. Uncheck "master" when installing. (If you didn't use the customized image)
